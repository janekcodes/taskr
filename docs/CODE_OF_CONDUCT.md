# 📜 Code of Conduct

This project is built around a simple idea: keep things respectful, focused, and constructive.

Taskr is currently a personal project, but I want to maintain a standard that keeps the environment clean and professional if others choose to contribute.

---

## 📌 My Expectations

Anyone interacting with this project should:

- Be respectful in all communication  
- Stay constructive when giving feedback  
- Focus on improving the project, not criticizing the person  
- Keep discussions relevant to the project goals  

---

## 🧠 Project Mindset

Taskr is built around simplicity, performance, and control.

That mindset also applies to communication:

- Keep feedback clear and useful  
- Avoid unnecessary negativity  
- Respect design decisions, even if you disagree  
- Focus on improvement, not argument  

---

## 🚫 Unacceptable Behavior

The following behaviors are not acceptable:

- Disrespectful or toxic language  
- Personal attacks  
- Spam or irrelevant issues  
- Disruptive behavior in pull requests or discussions  
- Attempts to intentionally break or misuse the project  

---

## 🔧 Contribution Respect

If you contribute:

- Respect the existing architecture and design direction  
- Avoid large unrelated changes in a single PR  
- Be open to feedback and iteration  
- Understand that not all contributions may be accepted  

---

## 🧠 Decision Authority

Since this is currently a solo-built project, final decisions regarding:

- architecture  
- features  
- code structure  
- direction  

are made by the project maintainer.

This is to ensure consistency and long-term stability of the system.

---

## 📌 Goal of This Codebase

The goal of Taskr is not just to build software, but to build something:

- clean  
- minimal  
- fast  
- and intentionally designed  

All contributions and interactions should support that direction.

---

## 🚀 Summary

Be respectful.  
Stay constructive.  
Keep it focused.

That’s it.
